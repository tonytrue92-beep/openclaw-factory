# ТЗ на Воркбук «Установка и настройка OpenClaw»

**Дата**: 2026-04-16
**Заказчик**: Антон Поляков
**Источник материалов**: репозиторий [tonytrue92-beep/openclaw-factory](https://github.com/tonytrue92-beep/openclaw-factory)
**Формат выдачи**: PDF + редактируемый Notion/Google Docs

---

## 1. Цель воркбука

Провести ученика курса от нуля до рабочего Telegram-бота на OpenClaw **без привлечения поддержки**. Ученик открывает воркбук параллельно с установщиком, отмечает галочки, видит скриншоты, при проблеме находит решение сам.

Воркбук не заменяет видеоурок — он идёт вместе с ним и служит «страховочным тросом» и «памяткой после установки».

---

## 2. Целевая аудитория

- Предприниматели и маркетологи, **не программисты**
- macOS / Windows / Linux (в порядке приоритета)
- Уровень техники: умеют открывать терминал по инструкции, копипастить команды, создавать бота в BotFather
- Не знают, что такое gateway, DM policy, allowlist, tool calls

**Язык**: русский. Без англицизмов там, где можно без них. Английские термины (gateway, allowlist, token) — оставляем как есть, но поясняем на полях.

---

## 3. Структура воркбука

Воркбук делится на **4 больших блока**. Общий объём — ориентир 40–60 страниц А4 в PDF.

### Блок 1. Перед установкой (5–7 страниц)

- 1.1 Что такое OpenClaw — 1 абзац без технических деталей, схема «мессенджер → OpenClaw → модель → ответ»
- 1.2 Что получит ученик после прохождения — чек-лист результата
- 1.3 Требования:
  - macOS / Linux — просто Node.js 22.14+
  - Windows — WSL2 + Ubuntu (отдельная ссылка на Windows-инструкцию)
- 1.4 Что приготовить заранее:
  - Аккаунт на [opencode.ai](https://opencode.ai) + API-ключ
  - Telegram-бот через [@BotFather](https://t.me/BotFather) — токен
  - Свой Telegram ID через [@userinfobot](https://t.me/userinfobot)
- 1.5 Сколько времени займёт: ~15 минут для macOS/Linux, ~25 минут для Windows

**Что нужно от технаря**: ничего — текст + схема архитектуры (даём референсы).

---

### Блок 2. Прохождение установки (15–20 страниц)

Дублирует 6 шагов установщика (R1–R6) **со скриншотами на каждом шаге**.

| Шаг | Что происходит | Что видит ученик | Куда кликает / что вводит |
|-----|----------------|------------------|---------------------------|
| R0  | Запуск команды | Меню из 3 пунктов | Вводит `2` → Enter |
| R1  | Проверка системы | Node.js / npm версии | Ничего |
| R2  | Установка OpenClaw | Прогресс npm install | Ничего |
| R3  | OpenCode Zen → MiniMax Free | 8-шаговый гайд по opencode.ai | Копирует ключ |
| R4  | Telegram-бот | BotFather + userinfobot | Вставляет токен и ID |
| R5  | Создание ассистента | Прогресс | Имя агента |
| R6  | Финальная проверка | `/status` в Telegram | Отправляет `/status` |

**Каждый шаг = разворот страниц**:
- слева — скриншот терминала
- справа — объяснение на русском + галочка-чек-лист «я прошёл этот шаг»

Также на каждом шаге блок «Если упал» с самыми частыми ошибками именно этого шага (брать из `troubleshooting` установщика).

**Что нужно от технаря/дизайнера**: снять 15–20 скриншотов терминала на чистой macOS-VM с прохождением установщика. Скриншоты **реальные**, не фотошопные. Разрешение 2x для retina.

---

### Блок 3. После установки (8–10 страниц)

Дублирует блок «💡 ПОДСКАЗКИ» из установщика, но в удобочитаемом формате с примерами:

- 3.1 Как сменить модель
  - Быстрый путь: `openclaw-switch-model` (helper)
  - Интерактивное меню helper'а — скриншот
  - Таблица популярных моделей (бесплатные / платные / для чего использовать)
  - Ручной путь (4 команды) — для справки

- 3.2 Как изменить характер ассистента
  - Где лежат `IDENTITY.md` и `AGENTS.md`
  - Пример «до / после» правки (ассистент-строгий vs ассистент-дружелюбный)
  - Как применить изменения (sessions cleanup)

- 3.3 Как подключить второй канал
  - Список поддерживаемых (30+)
  - Команды для WhatsApp / Discord / Slack (с пометкой что где нужно)

- 3.4 Как завести второго агента
  - Пример: основной assistant + переводчик
  - Команды + скриншот результата

- 3.5 Где что хранится
  - Карта директорий `~/.openclaw/`
  - Как делать бэкап перед правками
  - Как восстановиться из бэкапа

**Что нужно от технаря**: 5–6 скриншотов результатов (Telegram-диалог, список моделей в CLI, `/status` с новой моделью).

---

### Блок 4. Если что-то сломалось (8–10 страниц)

Структура «симптом → причина → решение». Берём 10 пунктов troubleshooting из установщика + специфичные кейсы из [разбора клиентов](ii-team-topic10-errors-summary-2026-04-15.md):

| # | Симптом | На что смотреть | Что делать |
|---|---------|-----------------|------------|
| 1 | `zsh: command not found: openclaw` | Новый терминал | `export NVM_DIR=...` + `nvm.sh` |
| 2 | Бот пишет `access not configured` | DM policy | `openclaw pairing approve telegram <КОД>` |
| 3 | `brew not installed` при скиллах | Homebrew отсутствует | Ставим Homebrew |
| 4 | `npm error network ETIMEDOUT` | Сеть | Сменить DNS / VPN |
| 5 | Бот молчит, хотя всё «установлено» | `gateway.mode` пустой | `gateway.mode=local` → validate → restart |
| 6 | `onboard` виснет или циклится | Баг визарда | Ctrl+C → настройка через CLI |
| 7 | `Unknown model` / HTML вместо текста | Кривое имя модели | `openclaw models list --all` |
| 8 | Context overflow | Большая сессия | `openclaw sessions cleanup` |
| 9 | `401 No payment method` | Платная модель без биллинга | `openclaw-switch-model opencode/minimax-m2.5-free` |
| 10 | Хочу переустановить с нуля | Стейл-конфиг | `openclaw reset --scope config+creds+sessions` |

Каждый пункт = 1 страница. Скриншот ошибки + box «решение» с готовой командой.

В конце блока — **карточка-шпаргалка одной страницей**, которую можно вырвать/распечатать отдельно: «10 команд-выручалок».

**Что нужно от технаря**: 10 скриншотов конкретных ошибок. Некоторые придётся спровоцировать специально (например, закрыть Homebrew, сломать config, протухшая сессия).

## 4. Визуальные требования

### Общие
- Фирменный стиль OpenClaw / курса (logo, цветовая схема — Антон даст гайдлайн)
- Чёткая иерархия: H1 → H2 → H3
- Code-блоки моноширинным шрифтом с подсветкой (тёмная тема терминала, как в установщике)
- **Чек-боксы** на каждом шаге — ученик буквально ставит галочку карандашом

### Элементы
- 📸 Скриншоты терминала — рамка, подпись, номер шага
- 💡 Подсказки на полях — ёмкие, не больше 2 строк
- ⚠️ Предупреждения — красный фон, иконка
- ✅ Чек-листы в конце каждого блока
- 🔗 QR-коды для мобильных ссылок (opencode.ai, BotFather, userinfobot)

### Форматы выдачи
1. **PDF** — для распечатки и чтения в offline, A4, 300dpi
2. **Notion / Google Docs** — редактируемая версия, чтобы Антон мог править тексты без дизайнера
3. **Figma** — исходники дизайна, все слои редактируемые

## 7. Критерии готовности (acceptance)

Воркбук считается готовым, если:

- [ ] Новый ученик без нашей помощи проходит установку от начала до рабочего бота, следуя только воркбуку
- [ ] Все 10 типовых ошибок из разбора клиентов имеют решение в блоке 4
- [ ] Все скриншоты актуальны (установщик версии из `main` на момент съёмки)
- [ ] Версия установщика зафиксирована в футере каждой страницы («OpenClaw Factory v2026.04.x»)
- [ ] PDF корректно печатается на A4 без обрезов
- [ ] Notion-версия редактируется и не ломается при копировании таблиц
- [ ] Все ссылки кликабельны в PDF (opencode.ai, BotFather, документация)
