# 🌐 Установка OpenClaw на VPS сервер

**Для кого этот гайд**: ученики курса, которые хотят, чтобы Telegram-бот
работал **всегда** — 24/7, даже когда компьютер выключен, вы спите или
поехали в отпуск.

**Что понадобится**: VPS за 200-500 ₽/мес (или $5-10/мес), 20-30 минут
свободного времени, базовое умение копировать команды в терминал.

> Если технический бэкграунд ну совсем нулевой — не расстраивайтесь.
> Гайд написан так, что по нему пройдёт даже человек, который впервые
> слышит слово «SSH». Буквально копируйте команды.

---

## Что такое VPS простыми словами

VPS (Virtual Private Server) — это **удалённый компьютер в интернете**,
который вам сдают в аренду. Он работает круглосуточно, на нём установлен
Linux (обычно Ubuntu) и вы можете им управлять удалённо.

**Зачем оно**: ваш ноутбук выключается, закрывается крышкой, уходит
в сон — бот умирает вместе с ним. VPS всегда включён, бот на нём
работает непрерывно.

**Сколько стоит**: от 200 ₽/мес (в РФ) или от $5 (за рубежом). Дешевле,
чем одна чашка кофе в неделю.

> это мошенник, даже если представляется поддержкой.

## Шаг 2. Подключиться к VPS по SSH

SSH — это способ «войти» в удалённый компьютер из своего терминала.
Не бойтесь, всё через пару команд.

### На Mac

1. Откройте **Терминал**:
   - Нажмите **⌘ Cmd + пробел** → напечатайте `Терминал` → Enter
   - Или: Finder → Программы → Утилиты → Терминал

2. Введите (заменив IP на свой):
   ```bash
   ssh root@195.161.41.77
   ```

3. При первом подключении спросит: `Are you sure you want to continue connecting?`
   — ответьте **yes**, Enter.

4. Спросит **password** — введите пароль из письма провайдера.
   **⚠️ Символы не отображаются при вводе — это нормально**, просто
   печатайте и жмите Enter.

5. Если всё ок — вы увидите что-то вроде:
   ```
   Welcome to Ubuntu 22.04.3 LTS
   ...
   root@vps-1:~#
   ```

   Это значит вы **внутри VPS** — теперь все команды которые вы вводите,
   выполняются на сервере, не на вашем Mac.

### На Windows

1. Откройте **PowerShell**:
   - Нажмите **⊞ Win** → напечатайте `powershell` → Enter

2. Команды те же, что на Mac:
   ```powershell
   ssh root@195.161.41.77
   ```

3. Дальше так же: `yes` на первый вопрос → пароль → вы внутри.

> Если на Windows `ssh` говорит «команда не найдена» — откройте
> «Приложения и возможности» → «Дополнительные компоненты» →
> установите «Клиент OpenSSH». Или скачайте [PuTTY](https://putty.org/).

### Подсказка для обеих ОС

**Как понять что вы в SSH**: приглашение в терминале меняется на что-то
типа `root@vps-1:~#`. Если оно выглядит как `yourname@MacBook:~$` —
вы всё ещё на своём Mac.

**Как выйти из SSH**: введите `exit` и нажмите Enter. Это **не выключит
VPS** — сервер и бот продолжат работать. Просто вы «отключитесь».

## Шаг 3. Запустить установщик на VPS

Теперь вы в SSH. Скопируйте эту команду **целиком** и вставьте в
терминал (там же, где вы сейчас сидите):

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/tonytrue92-beep/openclaw-factory/main/scripts/demo-install.sh) --vps --install
```

**Что эта команда делает:**
- Скачивает наш установщик
- Запускает в режиме `--vps` (пропустит macOS-проверки, не будет пытаться
  открыть браузер, в конце даст VPS-инструкцию для dashboard)
- `--install` = сразу к установке без демо и меню выбора

**Дальше всё как на обычном компьютере:**

1. Установщик проверит систему (Node.js, npm) — при необходимости
   сам поставит через nvm, без вашего участия
2. Скачает OpenClaw
3. **Спросит API-ключ opencode.ai** — откройте https://opencode.ai
   в браузере, создайте ключ (`sk-...`), скопируйте, вставьте в терминал
4. **Спросит Telegram bot token** — если бота ещё нет, создайте через
   `@BotFather` в Telegram (`/newbot`), скопируйте токен, вставьте
5. **Спросит ваш Telegram user ID** — узнайте его через бота
   `@userinfobot` в Telegram
6. Через 3-5 минут бот будет работать

**Готово**. Напишите своему боту в Telegram — он ответит.

## Шаг 4. Отключиться, не потеряв бота

Когда всё проверили — просто введите в SSH:

```bash
exit
```

**Что происходит**: SSH-сессия закрывается, но бот остаётся работать.
Как это возможно? Установщик поднимает бота как **systemd-сервис** —
это «фоновая служба», которая:
- Стартует автоматически при включении VPS
- Перезапускается если упадёт
- Не зависит от того, залогинены вы по SSH или нет

Ваш ноутбук можно закрывать, Mac выключать — боту всё равно. Он
работает у Timeweb / Hetzner / где вы его купили, 24/7.

## Шаг 5. Открыть dashboard (опционально)

Если хочется посмотреть dashboard OpenClaw (там видно все сообщения,
агентов, каналы) — есть пара способов.

### Вариант А — SSH-туннель (простой и безопасный)

Dashboard работает на VPS только на `127.0.0.1:18789` — снаружи не
видно. Это безопасно: никто из интернета не может к нему достучаться.
Но вы можете «пробросить» порт через SSH:

**На вашем Mac** (в **новом** окне терминала, не в SSH-сессии):

```bash
ssh -L 18789:127.0.0.1:18789 root@<ip-вашего-vps>
```

Пока эта команда висит (не нажимайте Ctrl+C) — откройте
браузер и зайдите на:

```
http://127.0.0.1:18789
```

Это откроет dashboard вашего VPS-бота, как будто он локальный.

**Закрыть туннель**: в терминале с туннелем нажмите `Ctrl + C`.

### Вариант Б — не надо смотреть dashboard

Большинству клиентов dashboard не нужен. Работа проверяется проще:
напишите боту в Telegram → он ответил → всё в порядке.

### Вариант В — Reverse-proxy с HTTPS (для продвинутых)

Если хочется `https://ai.mycompany.com/dashboard` — нужен Caddy или
Nginx с SSL. Это не в этом гайде; спросите в саппорте курса.

## Что делать если что-то сломалось

### Бот замолчал / не отвечает

1. Подключитесь к VPS по SSH:
   ```bash
   ssh root@<ip>
   ```

2. Запустите диагностику:
   ```bash
   bash <(curl -fsSL https://raw.githubusercontent.com/tonytrue92-beep/openclaw-factory/main/scripts/demo-install.sh) --diagnose-only
   ```

   Покажет что работает, что сломано + готовые команды для починки.

3. Если 401 Invalid API key:
   ```bash
   openclaw-factory-reauth
   ```

4. Если не помогло — соберите debug-bundle и пришлите в саппорт:
   ```bash
   bash <(curl -fsSL https://raw.githubusercontent.com/tonytrue92-beep/openclaw-factory/main/scripts/demo-install.sh) --collect-debug
   ```

   Файл окажется на VPS в `~/openclaw-debug-*.zip`. Чтобы забрать на
   свой Mac (в **новом** окне терминала, не SSH):

   ```bash
   scp root@<ip>:/root/openclaw-debug-*.zip ~/Downloads/
   ```

### VPS перезагрузился и бот не поднялся автоматически

Не должно такого быть — systemd делает автозапуск. Но на всякий:

```bash
openclaw gateway start
openclaw gateway status
```

### Хочу переустановить всё с нуля

Просто запустите установщик снова:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/tonytrue92-beep/openclaw-factory/main/scripts/demo-install.sh) --vps --install
```

Он предложит меню «что делать с существующей установкой» — выберите
пункт 4 (полный сброс).

### Хочу поменять модель AI

Прямо на VPS:

```bash
openclaw-switch-model
```

Покажет интерактивное меню из 6 моделей + custom.

### Не хочу больше платить за VPS, хочу всё снести

1. Остановите сервис бота:
   ```bash
   openclaw gateway stop
   ```

2. Удалите OpenClaw с VPS:
   ```bash
   rm -rf ~/.openclaw
   npm uninstall -g openclaw
   ```

3. Зайдите в панель провайдера → удалите VPS.

## Часто задаваемые вопросы

### Нужен ли мне свой домен?
Нет. Бот работает через Telegram — домен не требуется.

### Можно ли использовать тот же API-ключ opencode.ai, что я использовал локально?
Да. Один ключ — несколько установок.

### А если я поставил локально, а теперь хочу на VPS — надо переносить?
Нет смысла. Поднимите новую установку на VPS (это 5 минут), напишите
боту из Telegram, проверьте что работает. Локальную установку можно
просто выключить — она не мешает.

### VPS сломался — что делать?
Провайдеры дают резервные копии (снапшоты). Но если резервных копий
нет — OpenClaw поднимается за 5 минут на любой новой машине. Никаких
критичных данных в нём не хранится.

### Могу ли я ставить на несколько VPS (backup)?
Физически можете, но Telegram-бот прикреплён к одному токену. Если
два бота с одним токеном будут пытаться отвечать — получится каша.
Лучше ставить на один VPS, плюс иметь резервные снапшоты у провайдера.

### Как обновить OpenClaw когда выйдет новая версия?
Пока вручную, по SSH:

```bash
npm install -g openclaw@2026.6.6
openclaw gateway restart
```

### А если у меня совсем-совсем нет технического бэкграунда?
Напишите в саппорт курса с темой «VPS-установка» — мы вас проведём.
Можем подключиться через видео-созвон и сделать вместе.

## Что-то не получилось

Это нормально. На VPS гораздо больше переменных, чем локально
(провайдер, сеть, фаервол, версия Ubuntu). Если что-то пошло не так —
**не удаляйте VPS сразу**, сначала попробуйте:

1. **Запустите диагностику на VPS**:
   ```bash
   bash <(curl -fsSL https://raw.githubusercontent.com/tonytrue92-beep/openclaw-factory/main/scripts/demo-install.sh) --diagnose-only
   ```

2. **Соберите debug-bundle и пришлите в саппорт**:
   ```bash
   bash <(curl -fsSL https://raw.githubusercontent.com/tonytrue92-beep/openclaw-factory/main/scripts/demo-install.sh) --collect-debug
   scp root@<ip>:/root/openclaw-debug-*.zip ~/Downloads/
   ```

3. Напишите в саппорт курса, приложите zip + опишите на каком шаге
   вы застряли.
